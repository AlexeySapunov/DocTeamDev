Определения
Terms
Публикация – опубликованная статья. В нашем случае это название статьи с указанием рубрики, автора и аннотация.

Незарегистрированный пользователь – неавторизованный пользователь, который не имеет учетной записи на ресурсе, может только просматривать публикации и зарегистрироваться.

Неавторизованный пользователь – пользователь, который имеет учетную запись, но не прошел аутентификацию и не вошел в систему.

Авторизованный пользователь – пользователь прошедший аутентификацию и авторизовавшийся в системе. Может просматривать публикации, размещать публикации, оставлять комментарии,
ставить лайки, отправлять сообщения другим пользователям, осуществлять другие действия согласно выданным ролям: зарегистрированный пользователь, модератор, администратор, описание ролей приведено в п.4.

Взаимодействие пользователя с ботом
User and bot interaction
1. Любой пользователь должен иметь доступ к кнопочному интерфейсу бота, чтобы взаимодействовать с ним.
Критерии:
         внизу экрана клавиатура бота с названиями разделов, кнопкой регистрации.
		 
2. Любой пользователь должен видеть последние N свежих публикаций на начальной странице, чтобы получить представление о ресурсе и предлагаемом контенте.
Критерии:
N=5;
публикации выдаются отсортированными по времени публикации в порядке от свежих к старым.

3. Публикация должна содержать заголовок, название раздела, к которому она принадлежит, краткую аннотацию, автора, чтобы из такого описания можно было понять, о чем эта публикация.
Критерии:
описание публикации содержит заголовок, автора, аннотацию, тематический раздел;
доступно всем пользователям просматривающим раздел.

4. Незарегистрированный пользователь должен иметь возможность создать учетную запись на ресурсе, чтобы в последующем иметь авторизированный доступ к функционалу ресурса, согласно выданным ролям.
Критерии:
при нажатии клавиши регистрации пользователь получает возможность ввести регистрационные данные: логин и пароль.

5. Зарегистрированный пользователь должен иметь возможность пройти аутентификацию, чтобы авторизоваться в системе.
Критерии:
пользователю предлагается ввести логин и пароль;
авторизованный пользователь получает возможность оставлять комментарии, ставить лайки, отправлять сообщения другим пользователям, создавать публикации, получает доступ к личному кабинету.

6. Авторизованный пользователь должен иметь доступ к своему личному кабинету, чтобы заполнить или изменить данные о себе, посмотреть свой рейтинг, количество лайков, просмотреть личные сообщения.
Критерии:
при нажатии клавиши Личный кабинет пользователь получает возможность заполнить информацию о себе: имя, пол, возраст, краткое описание;
в Личном кабинете отображается рейтинг, количество лайков и список личных сообщений;
при сохранении все введенные данные записываются в БД.

7. Пользователь должен иметь возможность выбрать интересующий его раздел, чтобы ознакомится с публикациями выбранной тематики.
Критерии:
         при нажатии на клавишу раздела в списке публикаций выводятся описания публикаций в установленном порядке.
		 
8. Пользователь должен иметь возможность просмотреть выбранную публикацию, чтобы ознакомиться с ней.
Критерии:
         при нажатии на кнопку Просмотреть выбранная публикация... (далее пока не ясно что?)
		 
9. Авторизованный пользователь нажимает кнопку Создать публикацию, чтобы опубликовать статью.
Критерии:
при нажатии кнопки Опубликовать пользователю предоставляется возможность ввести данные о публикации: заголовок, тематический раздел, аннотация, автор. Далее прикрепляется файл со статьей;
после заполнения всех данных публикация записывается в БД, ее можно увидеть в списке последних публикаций после его обновления.

10. Пользователь с ролью модератора должен иметь кнопку Скрыть  публикацию, чтобы публикация не отображалась в списке.
Критерии:
         при нажатии кнопки Скрыть выбранная публикация больше не отображается в списке.
		 
11. Пользователь с ролью модератора должен иметь кнопку Скрыть  комментарий, чтобы комментарий не отображалась в списке.
Критерии:
         при нажатии кнопки Скрыть выбранный комментарий больше не отображается в списке.
		 
12. Пользователь с ролью Администратора должен иметь функционал для  добавления модераторов, чтобы назначать модераторов.
Критерии:
         доступ к списку пользователей;
         кнопка Добавить;
         выбранному пользователю назначается роль Модератор.
		 
13. Пользователь с ролью Администратора должен иметь функционал для  удаления модераторов, чтобы отозвать полномочия модератора.
Критерии:
         доступ к списку модераторов;
         кнопка Удалить;
         у выбранного пользователя отзывается роль Модератор.
		 
14. Пользователь с ролью Администратора должен иметь доступ ко всем функциональным возможностям.
